Differences in Source Editor and Visual Editor

  1. Explain the difference between the source editor view and visual editor view in Rstudio

  • Source View is plain test (ASCII character will be expressed exactly as it is in the source view)

  • Visual View Renders the plain text and interprets it as Markdown text, it gives the text formatting text instead of the plain test.

echo - Include the source code in output

  1. Be able to set code chunk options per chunk, and/or for the whole document. Understand rules for precedence (which options will apply if both are set.)

So we can see that if we put echo in the very top it will take precedent of all R chunks but if we put it in the code chunk it will only do so in selected options”

  • Echo: If set to TRUE (default), the R code in the chunk will be printed in the output document. If set to FALSE, only the output (if any) will be printed.

  • Eval: If set to TRUE (default), the R code in the chunk will be evaluated and its output (if any) will be printed. If set to FALSE, the code will be skipped, but its text (if any) will be printed.

  • Error: If set to TRUE, errors generated by R while running the chunk will stop the rendering process and display an error message. If set to FALSE (default), errors will be treated as warnings and will not stop the rendering process. ”
